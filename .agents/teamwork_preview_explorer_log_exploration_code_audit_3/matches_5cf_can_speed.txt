Standard Operation: Commands Sent
Under normal operation, the receiver sends only **one** binary command to the VESC ESC to control the motor speed:

#### **`COMM_SET_DUTY` (Command ID: `5`)**
* **Purpose:** Sets the target duty cycle (motor speed/power percentage) of the ESC.
Scales the floating-point duty cycle to a 32-bit signed big-endian integer by multiplying by `100,000`:
     $$\text{val} = \text{duty} \times 100000$$
     *(e.g., maximum forward throttle $+100\%$ maps to a payload value of $+95,000$)*

#### **UART Packet Structure (Standard VESC Framing):**
The packet is transmitted as a sequential stream of 9 bytes:

| Byte Index | Field Name | Hex / Formula | Description |
| :--- | :--- | :--- | :--- |
| **0** | **Start Byte** | `0x02` | VESC start frame marker for payload size $\le 256$ bytes |
| **1** | **Payload Length** | `0x05` | Payload length is exactly 5 bytes |
| **2** | **Command ID** | `0x05` | `COMM_SET_DUTY` VESC command |
| **3** | **Data MSB** | `(val >> 24) & 0xFF` | Most significant byte of the 32-bit duty cycle integer |
| **4** | **Data Byte 2** | `(val >> 16) & 0xFF` | Second byte |
| **5** | **Data Byte 3** | `(val >> 8) & 0xFF` | Third byte |
| **6** | **Data LSB** | `val & 0xFF` | Least significant byte |
| **7** | **CRC MSB** | `(crc >> 8) & 0xFF` | CRC-16-CCITT (polynomial `0x1021`, initial `0`) calculated over bytes 2â€“6 |
| **8** | **CRC LSB** | `crc & 0xFF` | LSB of the calculated CRC-16 |
| **9** | **End Byte** | `0x03` | VESC end frame marker |
